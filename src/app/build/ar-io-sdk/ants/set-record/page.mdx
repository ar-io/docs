# setRecord

`setRecord` is a method on the `ANT` class that updates or creates a record in the ANT process.

`setRecord` requires authentication.

<Tip title="TTL">
  Time-To-Live (TTL) determines how often gateways should check the ANT for
  updated records. A record that is updated frequently should be set to a lower
  number to facilitate serving current data, while a record that is updated less
  often should be set to a higher number to allow cached data to be served more
  quickly.

  `ttlSeconds` must be set between the minimum of 60 seconds (1 minute) and the maximum of 3600 seconds (1 hour)
</Tip>

## Examples

<CodeGroup title="setRecord">

       ```typescript {{title: "NodeJS"}}
        const fs = require("fs");
        const { ANT, ArweaveSigner } = require("@ar.io/sdk");

        async function main() {
        const jwk = JSON.parse(fs.readFileSync("KeyFile.json"));
        const ant = ANT.init({
            signer: new ArweaveSigner(jwk),
            processId: "bh9l1cy0aksiL_x9M359faGzM_yjralacHIUo8_nQXM"
        });

        const { id: txId } = await ant.setRecord(
                {
                    undername: '@',
                    transactionId: '432l1cy0aksiL_x9M359faGzM_yjralacHIUo8_nQXM'
                    ttlSeconds: 3600
                },
                // optional additional tags
                { tags: [{ name: 'App-Name', value: 'My-Awesome-App' }] },
            );
        }

        main();

    ```
