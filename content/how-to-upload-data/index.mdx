# How to Upload Data to Arweave

Arweave offers **permanent data storage** - pay once, store forever. Choose the method that fits your needs.

## Upload Methods

<Cards>
  <Card href="/how-to-upload-data/no-code" title="No-Code Solutions">
    **Perfect for:** Non-technical users, one-off uploads, quick testing
    
    **Examples:** ArDrive, Arweave.app, AR.IO Gateway UI
    
    **Pros:**
    - No coding required
    - User-friendly interfaces
    - Quick setup and upload
    
    **Cons:**
    - Limited customization
    - Manual process
    - Not suitable for automation
  </Card>

  <Card
    href="/how-to-upload-data/bundling-services"
    title="Bundling Services â­ Recommended"
  >
    **Perfect for:** Most developers, production applications, cost optimization
    
    **Examples:** Irys, Arweave Bundler, AR.IO Bundler
    
    **Pros:**
    - Multiple payment options (AR, ETH, credit cards)
    - Optimized performance
    - Cost-effective for large uploads
    - Built-in retry mechanisms
    
    **Cons:**
    - Requires API integration
    - Dependency on third-party service
  </Card>

  <Card href="/how-to-upload-data/direct-to-network" title="Direct to Network">
    **Perfect for:** Advanced users, custom implementations, maximum control
    
    **Examples:** arweave-js, arweave-py, custom implementations
    
    **Pros:**
    - Full programmatic control
    - Native Arweave features
    - No third-party dependencies
    - Complete customization
    
    **Cons:**
    - Requires AR tokens
    - More complex implementation
    - Higher technical barrier
  </Card>
</Cards>

## Why Choose Arweave?

- **Permanent**: Data stored once, available forever
- **Decentralized**: No single point of failure
- **Cost-effective**: Pay once, store forever
- **Censorship-resistant**: No one can take it down
