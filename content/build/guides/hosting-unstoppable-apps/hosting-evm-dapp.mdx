## Introduction

This guide shows you how to build and deploy an Ethereum dApp frontend to Arweave using ArNS. You'll create a complete ETH wallet dashboard with balance checking and transfer functionality, then host it permanently on Arweave while your smart contract logic runs on Ethereum.

This **hybrid architecture** combines the best of both worlds:
- **EVM chains** (Ethereum, Base, etc.) for smart contract execution and state management
- **Arweave + ArNS** for permanent, censorship-resistant frontend hosting

By the end of this guide, you'll have a production-ready dApp that users can access through your ArNS name, with the frontend stored permanently on Arweave and transactions executing on Ethereum or Base.

<Callout title="New to unstoppable apps?" type="info">
Check out the [series introduction](/build/guides/hosting-unstoppable-apps) to understand how AR.IO Network makes apps truly unstoppable with 100+ independent gateways.
</Callout>

## What You'll Build

You'll create an **ETH Wallet Dashboard** that demonstrates core dApp functionality:

- **Multi-Wallet Support**: Connect with any browser extension wallet (MetaMask, Coinbase Wallet, Rainbow, Brave Wallet, etc.) plus mobile wallets via WalletConnect
- **Multi-Chain**: Support both Ethereum Mainnet and Base
- **View Balance**: Display user's ETH balance on connected chain
- **Send ETH**: Transfer ETH to other addresses with gas estimation
- **Network Switching**: Let users switch between Ethereum and Base
- **Permanent Hosting**: Deploy to Arweave with your ArNS domain

The complete app will be ~350 lines of code across 5 components, fully typed with TypeScript.

## Prerequisites

Before starting, ensure you have:

- **Node.js 18+** installed
- **Ethereum Wallet** (MetaMask, Coinbase Wallet, or similar) - You'll use this for both testing your dApp AND deploying to Arweave
- **ETH on Base** for Arweave storage payments (or Turbo credits from [turbo.ar.io/topup](https://turbo.ar.io/topup))
- **ArNS Name** configured and ready ([purchase](https://arns.app))
- Basic understanding of React and Ethereum

<Callout title="One Wallet for Everything" type="info">
This guide uses your Ethereum wallet (MetaMask, etc.) for both:
- **Testing your dApp** - Connect to your dashboard, view balance, send transactions
- **Deploying to Arweave** - Use your wallet's private key with `permaweb-deploy`

You can pay for Arweave storage with Base-ETH tokens directly from your Ethereum wallet, making this a fully EVM-native workflow.
</Callout>

<Callout title="Why RainbowKit?" type="info">
This guide uses [RainbowKit](https://rainbowkit.com) for wallet connections - a production-ready library that provides beautiful UI and supports 100+ wallets out of the box. </Callout>

### Step 1: Configure RainbowKit

Create `src/wagmi.config.ts` to configure RainbowKit with your supported chains:

```typescript title="src/wagmi.config.ts"
import { connectorsForWallets } from '@rainbow-me/rainbowkit';
import { walletConnectWallet } from '@rainbow-me/rainbowkit/wallets';
import { createConfig, http } from 'wagmi';
import { mainnet, base } from 'wagmi/chains';

const connectors = connectorsForWallets(
  [
    {
      groupName: 'Other',
      wallets: [walletConnectWallet],
    },
  ],
  {
    appName: 'ETH Dashboard',
    projectId: 'YOUR_WALLETCONNECT_PROJECT_ID',
  }
);

// Configure RainbowKit with wagmi
export const config = createConfig({
  connectors,
  chains: [mainnet, base],
  transports: {
    [mainnet.id]: http(),
    [base.id]: http(),
  },
});
```

<Callout title="Why This Minimal Configuration?" type="info">
RainbowKit automatically detects browser extension wallets (MetaMask, Coinbase, Rainbow, Brave Wallet, etc.) and displays them in an "Installed" section - no manual configuration needed. We only explicitly add `walletConnectWallet` for mobile wallet support via QR code. This approach saves ~545KB compared to using `getDefaultConfig`, which bundles the MetaMask SDK and other wallet-specific code you don't need.
</Callout>

**What this configuration provides:**
- Automatic detection of all browser extension wallets
- WalletConnect v2 support for mobile wallets via QR code
- Minimal bundle size (~2KB vs 545KB+)
- Support for 100+ wallets without bundling wallet-specific code

<Callout title="Get Your Project ID" type="info">
Sign up at [cloud.walletconnect.com](https://cloud.walletconnect.com) for a free project ID. This unlocks support for 100+ wallets and enables mobile wallet connections via WalletConnect.
</Callout>

```typescript title="src/components/EthBalance.tsx"
import { useAccount, useBalance } from 'wagmi';
import { formatEther } from 'viem';

export function EthBalance() {
  const { address, isConnected, chain } = useAccount();

  // Fetch ETH balance for connected address
  const { data: balance, isError, isLoading } = useBalance({

  });

  if (!isConnected) {
    return (
      <div className="balance-card">
        <h2>üí∞ Your Balance</h2>

        <h2>üí∞ Your Balance</h2>

      </div>
    );
  }

  if (isError || !balance) {
    return (
      <div className="balance-card">
        <h2>üí∞ Your Balance</h2>
        <p className="error">Failed to load balance</p>
      </div>
    );
  }

  return (
    <div className="balance-card">
      <h2>üí∞ Your Balance</h2>
      <div className="balance-amount">
        {parseFloat(formatEther(balance.value)).toFixed(4)} {balance.symbol}
      </div>
      <div className="balance-details">
        <span className="chain-name">{chain?.name}</span>
        <span className="address-short">
          {address?.slice(0, 6)}...{address?.slice(-4)}
        </span>
      </div>
    </div>
  );
}
```

**Key concepts:**
- `useAccount()` - Get connected wallet address and chain
- `useBalance()` - Fetch ETH balance (auto-updates on transactions)

### Component 2: Send ETH

Allow users to send ETH to other addresses with proper gas estimation and error handling.

<Tabs items={['Vite', 'Next.js']}>
<Tab value="Vite">

Create `src/components/SendEth.tsx`:

```typescript title="src/components/SendEth.tsx"
import { useState } from 'react';
import { useAccount, useSendTransaction, useWaitForTransactionReceipt } from 'wagmi';
import { parseEther } from 'viem';

export function SendEth() {
  const { address, isConnected } = useAccount();
  const [recipient, setRecipient] = useState('');
  const [amount, setAmount] = useState('');

  // Hook to send transaction
  const {
    data: hash,
    isPending,
    sendTransaction,
    error
  } = useSendTransaction();

  // Hook to wait for confirmation
  const { isLoading: isConfirming, isSuccess } = useWaitForTransactionReceipt({
    hash,
  });

  const handleSend = async (e: React.FormEvent) => {
    e.preventDefault();

    if (!recipient || !amount) {
      alert('Please enter recipient address and amount');
      return;
    }

    try {
      // Send ETH transaction
      sendTransaction({
        to: recipient as `0x${string}`,
        value: parseEther(amount), // Convert ETH to wei
      });
    } catch (err) {
      console.error('Transaction error:', err);
    }
  };

  if (!isConnected) {
    return (
      <div className="send-card">
        <h2>üì§ Send ETH</h2>
        <p>Connect your wallet to send ETH</p>
      </div>
    );
  }

  return (
    <div className="send-card">
      <h2>üì§ Send ETH</h2>

      <form onSubmit={handleSend}>
        <div className="form-group">
          <label htmlFor="recipient">Recipient Address</label>
          <input
            id="recipient"
            type="text"
            placeholder="0x..."
            value={recipient}
            onChange={(e) => setRecipient(e.target.value)}
            disabled={isPending || isConfirming}
          />
        </div>

        <div className="form-group">
          <label htmlFor="amount">Amount (ETH)</label>
          <input
            id="amount"
            type="number"
            step="0.001"
            placeholder="0.1"
            value={amount}
            onChange={(e) => setAmount(e.target.value)}
            disabled={isPending || isConfirming}
          />
        </div>

        <button
          type="submit"
          disabled={isPending || isConfirming}
        >
          {isPending ? 'Waiting for approval...' :
           isConfirming ? 'Confirming...' :
           'Send ETH'}
        </button>
      </form>

      {error && (
        <div className="error-message">
          Error: {error.message}
        </div>
      )}

      {hash && (
        <div className="transaction-hash">
          <p>Transaction Hash:</p>
          <code>{hash}</code>
        </div>
      )}

      {isSuccess && (
        <div className="success-message">
          ‚úÖ Transaction confirmed!
        </div>
      )}
    </div>
  );
}
```

**Key concepts:**
- `useSendTransaction()` - Send ETH transfers (triggers wallet approval)
- `useWaitForTransactionReceipt()` - Wait for transaction confirmation
- `parseEther()` - Convert ETH input to wei (1 ETH = 10^18 wei)
- Gas is estimated automatically by the wallet

## Deploy to Arweave Name System (ArNS)

### How Arweave Deployment Works

Unlike traditional hosting, deploying to Arweave means **permanent storage** with no servers to maintain. [`permaweb-deploy`](https://github.com/ar-io/permaweb-deploy) (already installed as a dev dependency) uploads your built files to Arweave via [Turbo](https://turbo.ar.io), then updates your ArNS name to point to the new deployment.

### What You'll Need

<Steps>

<Step title="ArNS Name">
A registered ArNS name from [arns.app](https://arns.app) - this becomes your permanent domain.
</Step>

<Step title="Payment">
Choose how to pay for permanent storage (~$0.10-0.50 for typical dApps):

- **Base-ETH** (recommended): Pay on-demand with ETH on Base network
- **Turbo Credits**: Pre-purchase at [turbo.ar.io/topup](https://turbo.ar.io/topup)

Use the [Turbo Calculator](https://turbo.ar.io/calculator) to estimate costs.
</Step>

<Step title="Private Key">
Export your MetaMask private key (used to sign the deployment transaction):

1. MetaMask extension ‚Üí Three dots ‚Üí "Account details"
2. "Show private key" ‚Üí Enter password ‚Üí "Confirm"
3. Copy the key (starts with `0x`)

<Callout title="Security Warning" type="warn">
Never share your private key or commit it to version control. Consider using a dedicated deployment wallet with minimal funds.
</Callout>
</Step>

</Steps>

### Setup Deployment Script

First, add a deployment script to your `package.json` for consistent deployments across local development and CI/CD:

<Tabs items={['Vite', 'Next.js']}>
<Tab value="Vite">

Add this script to your `package.json`:

```json title="package.json"
"scripts": {
  "deploy": "permaweb-deploy deploy --arns-name $ARNS_NAME --sig-type ethereum --private-key $PRIVATE_KEY --on-demand base-eth --max-token-amount 0.01 --ttl-seconds 60"
}
```

</Tab>
<Tab value="Next.js">

Add this script to your `package.json`:

```json title="package.json"
"scripts": {
  "deploy": "permaweb-deploy deploy --deploy-folder out --arns-name $ARNS_NAME --sig-type ethereum --private-key $PRIVATE_KEY --on-demand base-eth --max-token-amount 0.01 --ttl-seconds 60"
}
```

</Tab>
</Tabs>

**Key options explained:**
- `--deploy-folder out` - Build output folder (Next.js uses `out`; Vite uses default `dist`)
- `--arns-name $ARNS_NAME` - Your ArNS name (passed as environment variable)
- `--sig-type ethereum` - Use Ethereum wallet for signing
- `--private-key $PRIVATE_KEY` - Your wallet's private key (passed as environment variable)
- `--on-demand base-eth` - Pay for storage with Base-ETH
- `--max-token-amount 0.01` - Maximum spending limit per deployment
- `--ttl-seconds 60` - DNS updates propagate within 60 seconds

<Callout title="Why Use npm Scripts?" type="info">
Using `npm run deploy` instead of direct CLI commands ensures:
- **Consistency** - Same deployment configuration everywhere
- **Maintainability** - Update deployment settings in one place
- **Team collaboration** - Easier for team members to deploy
- **CI/CD integration** - Simpler workflow configurations
</Callout>

### Deploy Command

Now you can deploy with a simple command:

```bash
# Deploy with Base-ETH
ARNS_NAME=my-eth-dashboard PRIVATE_KEY="0xYOUR_PRIVATE_KEY" npm run deploy
```

<Callout title="Alternative: Turbo Credits" type="info">
If you prefer to use Turbo Credits instead of Base-ETH, modify your deploy script to remove the `--on-demand base-eth` and `--max-token-amount` flags:

**For Vite:**
```json
"deploy": "permaweb-deploy deploy --arns-name $ARNS_NAME --sig-type ethereum --private-key $PRIVATE_KEY --ttl-seconds 60"
```

**For Next.js:**
```json
"deploy": "permaweb-deploy deploy --deploy-folder out --arns-name $ARNS_NAME --sig-type ethereum --private-key $PRIVATE_KEY --ttl-seconds 60"
```
</Callout>

### Access Your dApp

After deployment completes (1-2 minutes), access your dApp at:

- `https://my-eth-dashboard.arweave.net`
- `https://my-eth-dashboard.ar.io`

Your ArNS name works as a subdomain on any AR.IO gateway! Discover 100+ available gateways at [gateways.ar.io](https://gateways.ar.io) where you can access your dApp the same way.

### Multi-Environment Setup

Deploy to different ArNS undernames for staging and production. First, add environment-specific deploy scripts to your `package.json`:

<Tabs items={['Vite', 'Next.js']}>
<Tab value="Vite">

```json title="package.json"
"scripts": {
  "deploy": "permaweb-deploy deploy --arns-name $ARNS_NAME --sig-type ethereum --private-key $PRIVATE_KEY --on-demand base-eth --max-token-amount 0.01 --ttl-seconds 60",
  "deploy:staging": "permaweb-deploy deploy --arns-name $ARNS_NAME --undername staging --sig-type ethereum --private-key $PRIVATE_KEY --on-demand base-eth --max-token-amount 0.01 --ttl-seconds 60",
  "deploy:testnet": "permaweb-deploy deploy --arns-name $ARNS_NAME --ario-process testnet --sig-type ethereum --private-key $PRIVATE_KEY --on-demand base-eth --max-token-amount 0.01 --ttl-seconds 60"
}
```

</Tab>
<Tab value="Next.js">

```json title="package.json"
"scripts": {
  "deploy": "permaweb-deploy deploy --deploy-folder out --arns-name $ARNS_NAME --sig-type ethereum --private-key $PRIVATE_KEY --on-demand base-eth --max-token-amount 0.01 --ttl-seconds 60",
  "deploy:staging": "permaweb-deploy deploy --deploy-folder out --arns-name $ARNS_NAME --undername staging --sig-type ethereum --private-key $PRIVATE_KEY --on-demand base-eth --max-token-amount 0.01 --ttl-seconds 60",
  "deploy:testnet": "permaweb-deploy deploy --deploy-folder out --arns-name $ARNS_NAME --ario-process testnet --sig-type ethereum --private-key $PRIVATE_KEY --on-demand base-eth --max-token-amount 0.01 --ttl-seconds 60"
}
```

</Tab>
</Tabs>

Then create a workflow that uses the appropriate script based on the branch:

```yaml title=".github/workflows/deploy.yml"
name: Deploy to Arweave

on:
  push:
    branches: [main, develop]
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build project
        run: npm run build

      - name: Deploy to Production
        if: github.ref == 'refs/heads/main'
        env:
          ARNS_NAME: ${{ secrets.ARNS_NAME }}
          PRIVATE_KEY: ${{ secrets.PRIVATE_KEY }}
        run: npm run deploy

      - name: Deploy to Staging
        if: github.ref == 'refs/heads/develop'
        env:
          ARNS_NAME: ${{ secrets.ARNS_NAME }}
          PRIVATE_KEY: ${{ secrets.PRIVATE_KEY }}
        run: npm run deploy:staging
```

This configuration deploys:
- `main` branch ‚Üí `myapp.arweave.net` (production)
- `develop` branch ‚Üí `staging_myapp.arweave.net` (staging, using `staging` undername)

The benefit of this approach is that deployment configuration lives in `package.json`, making it easy to test locally:
```bash
# Test staging deployment locally
ARNS_NAME=myapp PRIVATE_KEY="0x..." npm run deploy:staging

# Test production deployment locally
ARNS_NAME=myapp PRIVATE_KEY="0x..." npm run deploy
```
